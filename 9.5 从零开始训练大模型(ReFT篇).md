       input_ids = tokenizer.encode(text, return_tensors="pt", truncation=True, max_length=max_length).to(device)
       start_index = text.rfind(target_span)
       start_token = len(tokenizer.encode(text[:start_index]))
       end_token = input_ids.shape[1]
       labels = input_ids.clone()
       labels[0, :start_token] = -100
       with torch.no_grad():
           outputs = model(input_ids, labels=labels)
       loss = outputs.loss
       perplexity = torch.exp(loss)
       losses = []
       logits = outputs.logits
       for i in range(1, end_token):
           log_prob_dist = log_softmax(logits[0, i-1])
           true_token = input_ids[0, i]
           token_loss = nll_loss(log_prob_dist.unsqueeze(0), true_token.unsqueeze(0))
           losses.append(token_loss.item())
       return perplexity.to('cpu'), 0, losses

**ReFT训练**

> 训练数据集和SFT预热的一致，并使用PPO算法。其中，对于中间过程（st+1没有得到answer），奖励为0。对于得到了错误的answer（与y不等），奖励设为0.1。（对于answer全为数字的数据集，当answer可以从推理路径中提取且为数字型时，可以采用部分奖励partial reward为0.1，这是一种缓解奖励稀疏的方法）。

# **9.5.4 结果分析**

> 在评测集上的效果非常不理想，容易出现胡言乱语的现象以及指令遵循能力不佳，参考最近的DPSK R1的一些复现工作，有如下猜想：
>
> 1. Pretrain阶段加的Math类数据占比少，而直接用Math类的数据做`ReFT`，导致效果不佳。
>
> 2. 模型太小，TinyZero工作中也提到了，至少得3B以上的模型才会出现顿悟时刻，0.8B直接做ReFT显然不太够。
>
> 3. 训练数据集来源的单一，导致模型泛化能力不够

**下一步可做的点：**

1. 先用`Qwen2.5-1.5b-Math`和`Qwen2.5-1.5b`做一组对照试验来验证猜想一。

2. 用`Qwen2.5-0.5b`、`Qwen2.5-1.5b`和`Qwen2.5-7b`做一组对照试验来验证猜想二。

3. 收集多个中英文数据集做数据配比验证猜想三。
