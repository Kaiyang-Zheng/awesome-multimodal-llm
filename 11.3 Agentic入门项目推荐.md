## 一、理解 Agent 的核心机制

### 1. pguso/ai-agents-from-scratch （1.6k stars）

**项目简介：** 该项目旨在通过使用本地LLM（例如 node-llama-cpp）和基础编程语言。

**推荐理由：** 它的哲学是通过构建来学习，帮助开发者深入理解 Agents 的工作原理，如 ReAct 模式和功能调用的底层实现，避免了高级框架的复杂抽象,。

### 2. browser-use/browser-use  (73.2k stars)

**项目简介：** 这是一个开源库，赋予 LLM 智能体进行**网页浏览、交互和自动化**的能力，让 Agent 能够在网站上执行任务，如填写表格或购物。该项目旨在让网站对 AI Agents 变得可访问。

**推荐理由：** 该项目将抽象的“工具调用”概念转化为具体的**与外部世界的交互能力**，展示了 Agent 如何访问真实的、有状态的网站，并处理认证、会话保持等问题。它还支持使用 Ollama 运行本地 LLM。

## 二、Agent实战

### 1. TransformerOptimus/SuperAGI  (16.9k stars)

**项目简介：** 这是一个面向开发者的开源**自主 AI Agent 框架**，旨在帮助开发者快速、可靠地构建、管理和运行实用的自主智能体。

**推荐理由：** Dev-first 的生产化平台。 提供GUI，支持并发运行 Agent，并解决了生产部署中的关键工程问题，例如Performance Telemetry优化 Token 使用，这对于控制成本和监控 Agent 轨迹至关重要,。

### 2. crewAIInc/crewAI  (41k stars)

**项目简介：** CrewAI 是一个用于编排角色扮演自主 AI Agent 的框架。它通过明确定义 Agent Role、Goal和任务Task，并将其组织成协作流程，来促进协作智能，共同解决复杂任务,。

**推荐理由：** 最受欢迎的多 Agent 框架之一，上手难度适中，商业落地快。

### 3. TauricResearch/TradingAgents  (26.1k stars)

**项目简介：** 这是一个多 Agent 金融交易框架，模拟真实世界交易公司的运作动态。它部署了专业的 LLM 驱动 Agent 团队（如基本面分析师、情感分析师、风险管理团队），通过**动态讨论**共同评估市场并制定交易决策。

**推荐理由：** Agent-Based Modeling 的专业领域典&#x8303;**。**

### 4. kubernetes-sigs/agent-sandbox  (434 stars)

**项目简介：** 这是 CNCF（云原生计算基金会）的一个重要子项目，旨在为 Kubernetes 提供一个标准化、声明式的 Sandbox API，用于管理**孤立、有状态、单例工作负载**，特别适用于 **AI Agent 运行时**，以安全地执行 LLM 生成的不可信代码,。

**推荐理由：**  虽然 Star 数量不高（属于 SIGs 项目，价值不在流行度），但它解决了 Agent 在生产环境中的核心安全和可扩展性问题。
